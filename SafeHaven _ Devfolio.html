The problem SafeHaven solves

During natural disasters or emergency situations, there's a lack of real-time, organized coordination between victims, responders, and local authorities. Communication lines break down, misinformation spreads, and people struggle to find safe routes, shelters, or medical help.

SafeHaven solves this by acting as a centralized AI-powered platform that helps:

Civilians find the nearest safe zones, open hospitals, and live updates.

First responders plan optimized routes and access critical real-time data.

Authorities track movement, predict risks, and send verified alerts.

Families to check on their loved ones using low-bandwidth emergency messaging.

All this is accessible through a mobile app, web portal, and even a fallback SMS service for offline or no-internet scenarios.

Challenges we ran into

Real-time Data Ingestion:

  1. Integrating multiple APIs (weather, government, satellite) and syncing them efficiently was tough. We solved this using a message queue system (Kafka) and real-time event-driven architecture.
    Offline Functionality:
  2. Ensuring users could get alerts even with no internet was tricky. We built an SMS-based fallback system and lightened the core mobile app using Flutters minimal build configuration.
    Routing During Disasters:
  3. egular GPS APIs didnt work well in blocked routes or flooded areas. We had to train a custom AI model on historical disaster maps to offer smarter alternate routes based on terrain and past incidents.
    Fake Alert Prevention:
  4. We faced issues with unverified sources spamming the app. We added a moderation layer using NLP-based filtering and crowdsourced reputation scoring.
